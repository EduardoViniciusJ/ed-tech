**Tipo:** Portal
**Prioridade**: Alta  
**Update Set**: EDTECH_05_PORTAL_V2_STRY0065_HOME_ALUNO  
**Parent**: EDTECH_05_PORTAL_V2

**Persona**:  
Aluno que acessa o portal EdTech e precisa cair diretamente em uma página inicial simples, organizada e voltada para abertura e acompanhamento de solicitações acadêmicas.

**O que precisa**:  
Criar a página inicial do aluno no Service Portal e vinculá-la como **Homepage**do portal EdTech.

**Escopo desta story**:

- Criar a página edtech_home_aluno.
- Preencher título e descrição da página.
- Vincular a página como homepage do portal EdTech.
- Manter a página protegida para usuário logado.
- Preparar a página para receber os widgets futuros da Home do Aluno.

**Onde aplicar**:

- Service Portal > Pages
- Service Portal > Portals
- Portal: EdTech
- Página: edtech_home_aluno

**O que fazer no ServiceNow**:

- Criar nova página em Service Portal > Pages.
- Preencher:
    - **Title**: EdTech Home Aluno
    - **ID**: edtech_home_aluno
    - **Short description**: Página inicial do aluno no portal EdTech, usada para acesso rápido a solicitações, aulas e acompanhamento de pedidos.
    - **Public**: false
    - **Draft**: false
- Abrir o portal EdTech.
- Configurar:
    - **URL suffix**: edtech
    - **Homepage**: edtech_home_aluno
    - **404 page**: 404
- Validar que o portal abre a página correta ao acessar /edtech.

**Critérios de aceite**:

- A página edtech_home_aluno foi criada.
- A página possui título e descrição preenchidos.
- O portal EdTech usa edtech_home_aluno como homepage.
- Ao acessar /edtech, o aluno é direcionado para a Home do Aluno.
- A página não está pública para usuários sem login.
- A página fica pronta para receber os widgets da Home do Aluno.

**Tipo:** Portal
**Prioridade**: Alta  
**Update Set**: EDTECH_05_PORTAL_V2_STRY0068_WIDGET_PERFIL_ACADEMICO  
**Parent**: EDTECH_05_PORTAL_V2

**Persona**:  
Aluno ou professor que acessa o portal EdTech e precisa visualizar rapidamente seus dados acadêmicos principais em páginas diferentes do portal.

**O que precisa**:  
Criar um widget customizado e reutilizável para exibir os dados do perfil acadêmico do usuário logado, seguindo como base do wireframe da Home do Aluno. OBS: Não precisa ficar igual

**Escopo desta story**:

- Criar o widget EdTech - Perfil Acadêmico.
- Permitir reutilização do widget em várias páginas do portal.
- Buscar dados do usuário logado na tabela u_edu_perfil_academico.
- Buscar nome e iniciais na tabela sys_user.
- Exibir dados em formato de banner/hero.
- Não permitir edição dos dados pelo widget.

**Onde aplicar**:

- Service Portal > Widgets
- Tabela u_edu_perfil_academico
- Tabela sys_user
- Página edtech_home_aluno
- Futuras páginas do portal EdTech

**Dados exibidos no widget**:

- Nome do usuário
- Iniciais do usuário
- Curso
- RA, quando for aluno
-Registro funcional, quando for professor/colaborador
- Camp us
- Período
- Tipo de perfil

**O que fazer no ServiceNow **:

- Criar widget chamado EdTech - Perfil Acadêmico.
- No **Server Script**, identificar o usuário logado com gs.getUserID().
- Consultar a tabela u_edu_perfil_academico filtrando pelo campo Usuário.
- Buscar os dados acadêmicos/profissionais do usuário.
- Usar getDisplayValue() para campos de referência como Curso e Campus.
- Montar objeto data.usuario com nome, iniciais, tipo de perfil, RA/registro funcional, curso, campus e período.
- No HTML do widget, montar o banner visual com:
    - círculo de iniciais Obs: Use um Script que pegue o full name  e faça o split das iniciais. Podemos utilizar uma foto aqui tbm nesse circulo 
    - saudação Obs: Acho interessante ter 
    - curso ou vínculo principal; Obs: Para aluno Curso, Campus, Periodo
    - RA
- Criar CSS próprio do widget para seguir o wireframe.
- Adicionar o widget na página edtech_home_aluno.

**Critérios de aceite**:

- Widget EdTech - Perfil Acadêmico foi criado.
- Widget exibe nome, iniciais, curso, RA, campus e período para aluno.
- Widget exibe nome, iniciais, curso/campus e registro funcional quando aplicável.
- Widget busca os dados em u_edu_perfil_academico.
- Widget usa dados do usuário logado, sem permitir escolher outro usuário manualmente.
- Widget pode ser inserido em mais de uma página do portal EdTech.
- Se o usuário não tiver perfil acadêmico, o widget exibe mensagem simples sem quebrar a página.

**O que precisa**:  
Atualizar os scripts dos Record Producers da Secretaria para copiar os dados dos Variable Sets para os campos reais da tabela u_edu_secretaria, incluindo a padronização de u_solicitante e opened_by, assim como já foi feito nos Record Producers financeiros.

**Record Producers impactados**:

- Solicitar Histórico Escolar
- Solicitar Diploma
- Emitir Comprovante de Matrícula
- Solicitar Declaração Acadêmica

**Onde aplicar**:

- Service Catalog > Record Producers
- Tabela destino: Solicitação Secretaria [u_edu_secretaria]
- Campo Script dos quatro Record Producers da Secretaria
- Variable Sets:
    - Dados do Solicitante
    - Dados Acadêmicos do Solicitante
    - Contato e Retorno
    - Descrição da Solicitação
    - Detalhes da Solicitação Acadêmica

**O que fazer no ServiceNow**:

- Abrir cada Record Producer da Secretaria.
- Validar que todos apontam para a tabela u_edu_secretaria.
- Validar que os Variable Sets corretos estão vinculados.
- Atualizar o campo Script dos quatro Record Producers.
- Adicionar a padronização de solicitante:

`var solicitante = producer.solicitante ? producer.solicitante.toString() : gs.getUserID(); current.u_solicitante = solicitante; current.opened_by = solicitante;`

- Mapear os dados acadêmicos para os campos da tabela.
