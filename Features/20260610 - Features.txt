## STRY0121 - Exibir Solicitação de Diploma Apenas para Alunos Formados

**Prioridade**: Alta  
**Update Set**: EDTECH_04_SECRETARIA_STRY0121_VISIBILIDADE_DIPLOMA  
**Parent**: EDTECH_04_SECRETARIA

**Objetivo**:  
Exibir o Record Producer Solicitar Diploma apenas para alunos que possuem curso concluído no Perfil Acadêmico.

**Onde aplicar**:

- Tabela Perfil Acadêmico
- Record Producer Solicitar Diploma
- User Criteria

**O que fazer**:

- Criar campo no Perfil Acadêmico:
    
    - **Label**: Curso concluído
    - **Name**: u_curso_concluido
    - **Type**: True/False
    - **Default**: false
- Adicionar o campo no formulário interno do Perfil Acadêmico.
    
- Preencher o campo nos alunos de teste:
    
    - true para alunos formados;
    - false para alunos ainda cursando.
- Criar User Criteria:
    
    - **Name**: EDTech - Aluno com curso concluído
    - **Advanced**: true
    - Regra: retornar verdadeiro apenas quando o usuário logado tiver Perfil Acadêmico com Curso concluído = true.
- Aplicar o User Criteria no Record Producer Solicitar Diploma, em **Available For**.
    

**Critérios de aceite**:

- Aluno com Curso concluído = true visualiza o item Solicitar Diploma.
- Aluno com Curso concluído = false não visualiza o item Solicitar Diploma.
- O controle funciona no portal com login real, sem impersonate.
- A configuração não altera os demais Record Producers da Secretaria.

## STRY0122 - Validar E-mail e Telefone no Variable Set Contato e Retorno

**Prioridade**: Média  
**Update Set**: EDTECH_05_PORTAL_STRY0122_VALIDAR_CONTATO_RETORNO  
**Parent**: EDTECH_05_PORTAL

**Objetivo**:  
Validar o formato do e-mail e do telefone informados pelo usuário nos formulários do portal que utilizam o Variable Set Contato e Retorno.

**Onde aplicar**:

- Variable Set Contato e Retorno
- Catalog Client Script

**O que fazer**:

- Criar um Catalog Client Script no Variable Set Contato e Retorno.
- Configurar o script como onSubmit.
- Identificar os nomes reais das variáveis de e-mail e telefone.
- Validar se o e-mail informado possui formato válido.
- Validar se o telefone possui DDD e quantidade mínima de dígitos.
- Bloquear o envio do formulário quando e-mail ou telefone estiverem inválidos.
- Exibir mensagem clara no campo correspondente.

**Mensagens sugeridas**:

- Informe um e-mail válido.
- Informe um telefone válido com DDD.

**Critérios de aceite**:

- Formulários que usam Contato e Retorno validam e-mail e telefone antes do envio.
- E-mails válidos permitem o envio.
- Telefones válidos com DDD permitem o envio.
- E-mails inválidos bloqueiam o envio.
- Telefones inválidos bloqueiam o envio.
- A mensagem de erro aparece no campo correspondente.
- A validação funciona no portal, sem depender de impersonate.
