- Criar estrutura própria para vínculos acadêmicos do professor.
- Preparar os dados que serão exibidos na Home do Professor.
- Não criar Catalog Item ou Record Producer novo.
- Não criar portal interno para professor.

- Validar/criar tabela u_edu_turma.
- Criar tabela u_edu_professor_turma.
- Criar tabela u_edu_aula.
- Criar tabela u_edu_atividade.

**Campos em u_edu_disciplina**:

- Nome da disciplina
- Código
- Curso → referência para u_edu_curso
- Carga horária
- Status

**Campos em u_edu_turma**:

- Nome da turma
- Curso → referência para u_edu_curso
- Disciplina → referência para u_edu_disciplina
- Campus → referência para u_edu_campus
- Turno
- Período letivo
- Quantidade de alunos
- Status

**Campos em u_edu_professor_turma**:

- Professor → referência para sys_user
- Turma → referência para u_edu_turma
- Curso → referência para u_edu_curso
- Disciplina → referência para u_edu_disciplina
- Campus → referência para u_edu_campus
- Turno
- Período letivo
- Status

**Campos em u_edu_aula**:

- Vínculo professor/turma → referência para u_edu_professor_turma
- Professor → referência para sys_user
- Turma → referência para u_edu_turma
- Disciplina → referência para u_edu_disciplina
- Data da aula
- Horário de início
- Horário de término
- Local ou link da aula
- Conteúdo da aula
- Status da aula

**Campos em u_edu_atividade**:

- Vínculo professor/turma → referência para u_edu_professor_turma
- Professor → referência para sys_user
- Turma → referência para u_edu_turma

**O que fazer no ServiceNow**:

- Criar ou copiar um que exista widget:
    - Nome: EDTech - Pesquisas Pendentes
    - ID: edtech_pesquisas_pendentes
- O widget deve identificar o usuário logado.
- O widget deve buscar pesquisas pendentes relacionadas ao usuário logado.
- O widget deve exibir:
    - Número da solicitação.
    - Área de atendimento.
    - Título/resumo da solicitação.
    - Data de encerramento.
    - Botão Responder pesquisa.
- O botão deve abrir a pesquisa correspondente.
- Após resposta, a pesquisa não deve aparecer mais como pendente.
- Inserir o widget na Home do Aluno.
- Inserir o widget na Home do Professor.
- Opcionalmente inserir no My Requests ou na página do ticket encerrado.

**Comportamento esperado**:

- Se não houver pesquisa pendente, o widget não aparece ou exibe estado vazio discreto.

- Se houver várias pesquisas pendentes, lista as mais recentes primeiro.
- Professor vê pesquisas de chamados que ele abriu.
- Aluno vê pesquisas das solicitações que ele abriu.

**Critérios de aceite**:

- Aluno visualiza pesquisas pendentes no portal.
- Professor visualiza pesquisas pendentes no portal.
- Usuário consegue abrir a pesquisa pelo botão do widget.
- Pesquisa respondida deixa de aparecer como pendente.
- Secretaria, Financeiro, Suporte e Gestor não visualizam esse widget no portal.
- O widget não exibe pesquisas de outro usuário.
- O widget não cria pesquisas duplicadas.

## STRY0089 - Criar Relatórios de Satisfação por Área e para o Gestor

**Prioridade**: Alta  
**Update Set**: EDTECH_01_CORE_STRY0089_RELATORIOS_SATISFACAO  
**Parent**: EDTECH_01_CORE_V2

**Persona**: Secretaria, Financeiro, Suporte e Gestor Acadêmico

**O que precisa**:  
Permitir que as áreas internas e o Gestor Acadêmico acompanhem os resultados das pesquisas de satisfação por área, serviço, nota média e comentários dos usuários.

**Escopo desta story**:

- Criar relatórios de satisfação por área.
- Permitir visão específica para Secretaria, Financeiro e Suporte.
- Permitir visão consolidada para Gestor Acadêmico.
- Não criar widget de resposta no portal.
- Não criar nova pesquisa nesta story; a pesquisa pertence à STRY0087.

**Onde aplicar**:

- Reports > Create New
- Self-Service > Dashboards
- Dashboard da Secretaria
- Dashboard do Financeiro
- Dashboard do Suporte
- Dashboard do Gestor Acadêmico


**O que fazer no ServiceNow**:

- Criar relatório Satisfação - Secretaria.
- Criar relatório Satisfação - Financeiro.
- Criar relatório Satisfação - Suporte.
- Criar relatório Satisfação - Geral EdTech.
- Exibir indicadores:
    - Nota média.
    - Quantidade de respostas.
    - Percentual de avaliações positivas.
    - Solicitações sem resposta de pesquisa.
    - Comentários recentes.
    - Satisfação por área.
    - Satisfação por tipo de serviço.
- Adicionar os relatórios nos dashboards correspondentes:
    - Dashboard - Secretaria Acadêmica
    - Dashboard - Financeiro Acadêmico
    - Dashboard - Suporte Técnico AVA
    - Dashboard - Gestor Acadêmico
- Garantir filtro por área:
    - Secretaria vê pesquisas da Secretaria.
    - Financeiro vê pesquisas do Financeiro.
    - Suporte vê pesquisas do Suporte.
    - Gestor vê visão consolidada.

**Critérios de aceite**:

- Secretaria consegue visualizar satisfação das solicitações acadêmicas.
- Financeiro consegue visualizar satisfação das solicitações financeiras.
- Suporte consegue visualizar satisfação dos chamados técnicos.
- Gestor consegue visualizar satisfação consolidada das áreas core.
- Os dashboards exibem nota média e quantidade de respostas.
- Comentários dos usuários podem ser consultados pelas áreas responsáveis.
- Usuários sem permissão não acessam dados de satisfação de outras áreas.
